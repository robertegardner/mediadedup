Matcher sensitivity

Controls how aggressively perceptual (video) and chromaprint (audio) matching groups files. Lower = more aggressive (more groups found). Takes effect on the next matcher run; existing groups are not affected until you re-run the matcher.

Existing duplicate groups

Auto-delete exact (sha256) duplicates

Files with identical bytes are unambiguous duplicates -- one keeper, the rest moved to trash. The keeper is the file with the longest path (most descriptive location), with size as a tiebreaker. Files are MOVED to .mediadedup-trash/; nothing is unlinked.

Auto-delete similarity matches above a threshold

For perceptual (video) and chromaprint (audio) groups, deletes everything above the chosen similarity score. Picks the highest-quality file as keeper: highest resolution × bitrate for video, highest bitrate × size for audio. Files are MOVED to .mediadedup-trash/.
higher = stricter (1.00 ≈ exact); 0.95 typical for confident dupes
